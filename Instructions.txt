**Football Universe Draft & Simulator - Cursor Instruction / Reference File**

**Project Goal**  
Build a fun, replayable, head-to-head football (soccer) team builder where players pick one fictional universe, draft 11 players from its 22-player squad, choose a formation, assign players to roles, then simulate matches against friends (async link sharing + live Supabase Realtime). Emphasize humor from mismatched fictional characters in a serious tactical football simulation.

### Core Rules & Constraints (Non-Negotiable)
- **No mixed universes** — one universe per team for thematic coherence and strategic choice.
- **Squads**: Use the exact balanced player lists below (with stats and Overall). Do not change names or drastically alter stats unless balancing is needed later.
- **Stats** (0-100): Pace, Power, Stamina, Tackling, Passing, GK.
- **Overall formula**: `(Pace×0.18 + Power×0.18 + Stamina×0.18 + Tackling×0.18 + Passing×0.18 + GK×0.10)`
- **Simulation**: Real-time feel. 60-second halves. One tick every 5 seconds (~12 ticks per half). Full stat-driven logic + randomness + commentary text on events (chances, goals, tackles, stamina issues, special triggers).
- **UI/UX Priority**: **Avoid all standard AI/vibe-coded aesthetics** (no default Tailwind indigo/blue cards, soft shadows, rounded everything, Inter font defaults). Make it feel like **live football broadcast graphics** (Sky Sports / ESPN style): dark backgrounds, glassmorphic or semi-transparent panels, high-contrast white/yellow text, team-color accents, scoreboard overlays, clean but dramatic.

### Visual Direction (Enforce This Strictly)
- **Color Palette** (define as CSS variables):
  - Background: `#0a0a0a` (deep stadium black)
  - Accents: Team-specific (e.g., gold for LOTR `#f59e0b`, red for Harry Potter `#b91c1c`, green for Mammals `#166534`)
  - Text: `#f1f5f9` (off-white), bright yellow `#eab308` for highlights
  - Panels: Dark glass (`bg-black/70` backdrop-blur-md, subtle borders `#334155`)
- **Fonts**: Import `Oswald` or `Impact` for headers (broadcast feel), `Roboto Mono` or system sans for data/stats.
- **Style**: Bold, high-contrast, scoreboard-like. Large player cards with position badges, animated stat bars, ticker-style commentary feed. Mobile-first responsive (vertical layout on phone feels like watching on a phone).
- **Polish Instructions**: Use broadcast-inspired animations (slide-ins, goal flash, red/yellow card pops). No generic neumorphism or minimal cards.

- CB: Pace 10%, Power 25%, Stamina 15%, Tackling 35%, Passing 15%, GK 0%

- CM: Pace 15%, Power 15%, Stamina 25%, Tackling 20%, Passing 25%
- DM: Pace 10%, Power 20%, Stamina 20%, Tackling 35%, Passing 15%

Formation dictates exact role slots (e.g., 4-4-2 = GK + 2 CB + 2 FB + 2 CM + 2 W + 2 ST).

### Match Simulation Logic
- 2 halves × 60 real-time seconds.
- Tick every 5 seconds.
- Per tick (alternating teams or simultaneous):
  1. Attacking team computes Attack Rating (weighted relevant stats of forwards/midfield).
  2. Defending team computes Defence Rating.
  3. Chance creation roll (Attack vs Defence + randomness).
  4. If chance: Shot resolution (Power + Passing vs GK + randomness) → Goal or save + commentary.
- Stamina: Drains per tick. Low base Stamina = faster degradation. Effective stats = base × (current stamina multiplier, min ~60%).
- Substitutions: Up to 5 total. Halftime (3 subs), post-goal window (1 sub).
- Commentary: Flavor text on every major event + character-specific specials (e.g., Gandalf “You shall not pass!” free tackle).
